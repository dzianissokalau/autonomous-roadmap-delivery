# Review Gate Prompt Fragment

After implementation and verification, write a fresh skeptical review artifact
before phase advancement. Advance only when acceptance criteria are satisfied,
required verification passed after the final change, the verdict is exactly
`delivered`, and roadmap/state/log/review evidence agree. If the verdict is
`needs-fix`, fix only current-phase findings and rerun verification. If the
verdict is `blocked`, record the blocker and stop.
